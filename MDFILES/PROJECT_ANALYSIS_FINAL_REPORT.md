## 🔍 মূল কারণ চিহ্নিতকরণ

### সমস্যা #১: কাস্টমার API - কঠোর লিমিট (১০০)

**ফাইল**: `convex/customers.ts` (লাইন ১৬)

```typescript
// ❌ আগে:
const limit = Math.min(args.limit || 20, 100);  // সর্বোচ্চ ১০০ কাস্টমার

// ✅ পরে:
const limit = Math.min(args.limit || 20, 10000);  // সর্বোচ্চ ১০,০০০ কাস্টমার
```

**কেন এটি সমস্যা ছিল**:
- ১০০+ কাস্টমার যুক্ত করা অসম্ভব
- Dashboard কাস্টমার সংখ্যা ভুল দেখায় (সবসময় ১০০ সীমিত)
- বড় ব্যবসায়ের জন্য অপর্যাপ্ত

---

### সমস্যা #২: সেলস API - নিম্ন ডিফল্ট লিমিট (৫০)

**ফাইল**: `convex/sales.ts` (লাইন ১৮)

```typescript
// ❌ আগে:
.take(args.limit || 50);  // শুধু ৫০টি সেলস ডিফল্ট

// ✅ পরে:
.take(args.limit || 1000);  // ১০০০টি সেলস ডিফল্ট
```

**কেন এটি সমস্যা ছিল**:
- Dashboard শুধু সর্বশেষ ৫০টি সেলস ডাটা পায়
- মাসের মোট বিক্রয় গণনা ভুল হয়
- রিপোর্টিং সিস্টেম অসম্পূর্ণ ডাটা নিয়ে কাজ করে

---

### সমস্যা #৩: Dashboard - কোন লিমিট পাঠাচ্ছে না

**ফাইল**: `src/components/Dashboard.tsx` (লাইন ৫০-৫৪)

```typescript
// ❌ আগে:
const salesData = useQuery(api.sales.list, {});  // কোন লিমিট নেই
const customersData = useQuery(api.customers.list, {});  // কোন লিমিট নেই

// ✅ পরে:
const salesData = useQuery(api.sales.list, { limit: 1000 });  // স্পষ্ট লিমিট
const customersData = useQuery(api.customers.list, { limit: 10000 });  // স্পষ্ট লিমিট
```

**কেন এটি সমস্যা ছিল**:
- API এর ডিফল্ট লিমিট খুব কম (৫০ এবং ১০০)
- Dashboard ইচ্ছাকৃতভাবে বড় সংখ্যা অনুরোধ করছে না
- ডাটা শো হয় কিন্তু সম্পূর্ণ নয়
